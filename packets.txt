  packet cycle:
    client:   hello           connect                             redirected/publish/subscribe/unsubscribe/disconnect/ping/pingack

    server:         challenge         connected/redirect/disconnect                                                     connected/publish/puback/suback/unsuback/disconect/ping/pingack

          where:
              - packet type: the type of packet(hello in this case). 1 byte [required]
              - protocol version: the version of the protocol this client uses. 1 byte [required]
              - client type: if this is the godot client, the cli tool, etc. 1 byte [required]
              - client version: version of this client. 1 byte
              - flags: for now, the last bit tells you if this is a session resume, and the previous bit tells you if this client is going to encrypt their payloads or not. 1 byte [required]
              - encryption: it tells you which encryption algorythm is going to be used in the payload. 1 byte [only present if encryptation flag is true]
              - session id: it tells you which session id this connection tries to resume. 8 bytes [only present if session resume flag is true]

      challenge:
        a bbbbbbbbbb...

        where:
          - a: the type of packet(challenge in this case). 1 byte [required]
          - b: the challenge the client has to solve in order to prove identity. it has 32 bytes [required]

        notes:
          - this packet has a fixed length of 33 bytes (packet type + challenge)

      connect:
        a bbbbbbbbbbb...

        where:
          - a: packet type, the type of packet(connect in this case). 1 byte [required]
          - b: the signature the client sent in response to the challenge. it has 64 bytes [required]

        notes:
          - this packet has a fixed length of 65 bytes
          - In the case of a redirect, also remember the redirected packet, which is the packet the client 
          will use to connect to the new server, and the redirected_ok packet, which is the packet the server 
          will use to provide the data to the client in the case of a valid redirect.

      redirect:
        a bbbbbbbb cccccccccccccc dddddddddddd

        where:
          - a: packet type (redirect in this case). 1 byte [required]
          - b: subscription id. the client has to send this to the new gateway in order to know
          that he is the one that has been redirected. it has 8 bytes [required]
          - c: the challenge the client has to sign in order to prove its identity. it has 32 bytes. [required]
          - d: a string that contains the path to the new gateway. it has a variable length [required]

      disconnect:
        a b

        where:
          - a: packet type (disconnect in this case). 1 byte [required]
          - b: disconnect reason. a byte that indicates the reason for the disconnection [required]

        notes:
          - this packet has a fixed length of 2 bytes

      connected:
        a bbbbbbbb c ch dddd eeeeeeee f gggggggg h i j k l m

        where:
          - a: packet type (connected in this case) 1 byte [required]
          - b: session id. 8 bytes [required]
          - c(25): encryption key flag. if present, value must be 25. 1 byte
          - ch: encryption algorithm. it appears only if encryption key flag is present. it tells you which algorithm is going to be used to encrypt payloads. it has 1 byte
          - d: encryption key size. it only is present if the encryption key flag is present. it tells you how many bytes has the key. it has 4 bytes.
          - e: encryption key. the key the client will use to encrypt payloads. it will accompany the encryption key size. it has the size specified in encryption
          - f(13): timeout flag. if present, value must be 13. 1 byte
          - g: timeout. it only appears if the timeout flag is present. it tells you the session timeout in milliseconds. it has 8 bytes
          - h(48): protocol version flag. if present, it must be 48. 1 byte
          - i: protocol version. it only appears if the protocol version flag appears. it tells you the protocol version to be used when communicating with the gateway. it has 1 byte
          - j(31): client type flag. if present, it must be 31. 1 byte
          - k: client type. it only appear if client type flag is present. it tells you what kind of client is this one. 1 byte
          - l(82): client version flag. if present, it must be 82. 1 byte
          - m: client version. it only appear next to client version flag. it tells you the version of the client. 1 byte

      redirected:
        a bbbbbbbb cccccccccccccc

        where:
          - a: packet type (redirected in this case). 1 byte [required]
          - b: subscription id. the id used to acces the subscription in the new server. it has 8 bytes [required]
          - c: the signature used to prove its identity. it has 64 bytes [required]
